5/* Cache for getcpu() to speed it up. Results might be a short time
6 out of date, but will be faster.
7
8 User programs should not refer to the contents of this structure.
9 I repeat they should not refer to it. If they do they will break
10 in future kernels.
11
12 It is only a private cache for vgetcpu(). It will change in future kernels.
13 The user program must store this information per thread (__thread)
14 If you want 100% accurate information pass NULL instead. */
15struct getcpu_cache {
16 unsigned long blob[128 / sizeof(long)];
17};
